feat(gbrain): surface init outcome in diagnostics (Sprint 2.2.5b)

Sprint 2.2.5a added a 120s timeout to `ensure_bundled_gbrain_initialized`,
but init failures still only show up in tracing logs. When init times
out / fails / can't find the bundle, the user sees "gbrain — 未连接"
in Settings → 系统 with no hint about how to recover. They have to
tail logs.

This commit captures the init outcome in AppState and renders it as
a new row in the System tab.

## Backend

`McpManager` adds `GbrainInitStatus` enum (mcp.rs):

```rust
#[serde(tag = "status", rename_all = "snake_case")]
pub enum GbrainInitStatus {
    NotAttempted,
    InProgress,
    Succeeded { duration_ms, at_ms },
    SkippedAlreadyInitialized { at_ms },
    Failed { error, stderr_tail, at_ms },
    BundleMissing,
}
```

`AppState` (app.rs) gains:

```rust
pub gbrain_init_status: Arc<Mutex<GbrainInitStatus>>
```

Initialized to `NotAttempted` at boot. Stage 3 flips it to
`InProgress` immediately before the init spawn, then to one of
`Succeeded` / `SkippedAlreadyInitialized` / `Failed` based on result.
The `BundleMissing` branch in Stage 3 (when `find_bun_path` /
`find_gbrain_entry` return None) writes that variant explicitly.

`SystemDiagnosticsReport.gbrain_init` (tauri_commands.rs) is a
clone of the current slot — `get_system_diagnostics` reads it at
the end of the existing memU+gbrain status collection.

Stage 3 init-failure handler now also extracts `stderr_tail` from
the error message (the `Err` arm of `ensure_bundled_gbrain_initialized`
already emits "stderr (last 20 lines):\n…" — we split on that marker
to populate the structured field separately from the summary).

## Frontend

`SystemTab.tsx` TS type `GbrainInitStatus` mirrors the Rust enum
shape (discriminated union by `status`). New `GbrainInitRow`
component renders below the gbrain BridgeCard when init was
attempted. Color + label + remediation hint per variant:

- `in_progress` → yellow pulsing dot, "首次启动 — PGlite 跑 ~63 次迁移"
- `succeeded` → green, shows duration
- `skipped_already_initialized` → green, "已初始化"
- `failed` → red, error text + actionable hint
  "运行 scripts/init-gbrain.sh 或删除 ~/.uclaw/gbrain/ 后重启"
- `bundle_missing` → red, "bundle 缺失" + hint
  "运行 scripts/setup-bun-runtime.sh + scripts/setup-gbrain-source.sh"
