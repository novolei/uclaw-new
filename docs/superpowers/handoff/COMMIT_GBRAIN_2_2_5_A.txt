fix(gbrain): wrap init in 120s timeout (Sprint 2.2.5a)

Sprint 2.1 init-fix added `ensure_bundled_gbrain_initialized` which
spawns `bun gbrain/cli.ts init --pglite --yes` synchronously at
Stage 3 boot. Normally PGLite's 63 migrations finish in 30-60s. If
the bun binary is corrupted, the install was partial, or PGLite
hits a deadlock, the boot hangs indefinitely — user can only
force-kill the app.

This commit:
- Converts `ensure_bundled_gbrain_initialized` to async.
- Replaces `std::process::Command` with `tokio::process::Command`
  + `kill_on_drop(true)` so dropping the future reaps the child.
- Wraps `cmd.output()` in `tokio::time::timeout(GBRAIN_INIT_TIMEOUT_SECS)`.
- Adds `pub const GBRAIN_INIT_TIMEOUT_SECS: u64 = 120` with rationale
  comment (2x typical max, with margin for slow disks).
- On timeout: returns Err with an actionable message pointing at
  `scripts/init-gbrain.sh` for manual recovery.
- main.rs Stage 3 call site adds `.await` (already in async-spawn
  context).

Existing idempotency contract (Ok(false) when PG_VERSION present)
preserved bit-for-bit — only the spawn path is now bounded.

Tests:
- `ensure_bundled_gbrain_short_circuits_when_already_initialized`
  (updated to #[tokio::test])
- `ensure_bundled_gbrain_returns_err_when_bun_not_executable` (new
  — verifies the spawn-failure path returns Err quickly, not via
  timeout)
- `timeout_pattern_kills_hung_process` (new — sanity-checks the
  underlying tokio::time::timeout + kill_on_drop pattern with
  /bin/sleep so the real function's correctness is independently
  exercised)

Verification (Mac side):
  # Temporarily rename bun binary to simulate corruption
  mv src-tauri/bunembed/bun src-tauri/bunembed/bun.bak
  rm -rf ~/.uclaw/gbrain/.gbrain/  # force re-init
  cargo tauri dev
  # → Stage 3 log: 'gbrain init failed — ...timed out after 120s'
  # → app continues to boot; gbrain not connected; no hang
  mv src-tauri/bunembed/bun.bak src-tauri/bunembed/bun  # restore

  # Or just run the unit tests:
  cd src-tauri && cargo test --lib mcp::gbrain_init_tests
