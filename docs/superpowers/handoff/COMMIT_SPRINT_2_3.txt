feat(learning): promote / demote facet IPC + UI (Sprint 2.3)

Closes the manual-override loop in the Settings → 学到的偏好 tab.
Sprint 2.2 made the FacetCache visible read-only; Sprint 2.3 gives
the user direct buttons to push a facet up to `active` or down to
`provisional` without going through a full rebuild.

Backend (src-tauri/):

- ipc.rs: +LearningPromoteFacetInput + LearningDemoteFacetInput DTOs
- tauri_commands.rs: +memory_learning_promote_facet + memory_learning_demote_facet,
  both routing through a shared `set_facet_state` helper that does
  one UPDATE on user_profile_facets + refreshes the FacetCache so
  the next prompt build sees the new state immediately.
- main.rs: register the 2 new commands in invoke_handler!
- 6 unit tests under `learning_set_state_sql_tests` cover:
    - promote: provisional → active
    - promote: forgotten → active (recovery path)
    - demote: active → provisional
    - dismiss: anything → forgotten
    - missing facet returns rows_updated=0 (no error)
    - idempotent on same target state (rows_updated=1, bumps updated_at)

Frontend (ui/):

- types.ts: +LearningPromoteFacetInput, LearningDemoteFacetInput,
  LearningSetStateOutcome (camelCase mirror of the wire shape)
- tauri-bridge.ts: +memoryLearningPromoteFacet + memoryLearningDemoteFacet
- LearnedProfileTab.tsx:
    - handlePromote + handleDemote with the same optimistic
      local-state-flip pattern as handleDismiss
    - FacetRow now renders a hover-revealed button cluster:
        - ChevronUp (promote) when state != 'active'
        - ChevronDown (demote) when state ∈ {active, provisional}
        - X (dismiss) when state != 'forgotten'
    - Buttons sit inside a `group-hover:opacity-100` wrapper so the
      row stays visually quiet at rest
- LearnedProfileTab.test.tsx: +3 cases
    - promote optimistically flips state to active
    - demote optimistically flips state to provisional
    - candidate / forgotten rows hide the demote button (and
      forgotten also hides dismiss)

Soft-override semantics (documented in IPC doc-comments + UI tooltips):
all three actions get re-evaluated by the next 30-min stability
rebuild. A user promote sticks only if evidence keeps the facet's
computed stability high enough — matches openhuman's "user choice is
signal, not a ceiling" design.

Verification: `npx tsc --noEmit` green in sandbox. cargo build +
cargo test + vitest must run Mac-side (sandbox blocked by missing
platform-specific rollup binary and no cargo).
