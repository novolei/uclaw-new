feat(learning): persist PROFILE.md to disk after each rebuild (Sprint 2.1a)

After the ProactiveService stability_detector rebuilds the FacetCache
(every 60 ticks = 30 min), it now also renders PROFILE.md from the
fresh cache and atomically writes it to disk at
`~/Documents/workground/brain/PROFILE.md`.

Why disk:
- gives the user a human-readable snapshot they can `cat`, diff,
  version-control, or hand-edit (managed blocks get overwritten on
  next rebuild; prelude/postlude are preserved across writes)
- mirrors the Phase 7.1+ pattern of brain dir being the on-disk
  source of truth + DB being the index

MemoryOsRuntimeConfig (`proactive/service.rs`):
- New field `profile_md_path: Option<PathBuf>` — `None` skips the
  write silently (CI, no Documents dir).

main.rs (Stage 3 boot):
- Resolves `BrainExportConfig::default_brain_root()
  .map(|root| root.join("PROFILE.md"))` and wires into runtime cfg
  alongside the existing `learning_scheduler` / `facet_cache`
  handoff.

Tick block (`proactive/service.rs`):
- After `FacetCache::refresh_from`, read existing PROFILE.md (to
  preserve user-edited prelude/postlude) -> render with the just-
  refreshed snapshots -> atomic write via `profile_md::write`. All
  IO errors are logged + swallowed; the cache stays valid even when
  disk is unwritable.
