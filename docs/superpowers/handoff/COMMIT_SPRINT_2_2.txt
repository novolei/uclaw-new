feat(learning): Profile UI Settings tab (Sprint 2.2)

Closes the visibility gap left by Sprint 1.10. Sprint 1 shipped the
`memory_learning_list_facets` / `memory_learning_dismiss_facet` /
`memory_learning_rebuild_now` IPCs but no frontend — Sprint 2 wired
the producer + consumer paths but still no UI. This commit makes the
openhuman FacetCache human-visible.

New: a 'learnedProfile' tab in Settings ("学到的偏好") rendered between
'memoryRecall' and 'imChannels' in the 核心 group. The tab shows:

- explanatory subtitle (what the cache is + where PROFILE.md lives
  on disk)
- 'active' / 'provisional' counts as outline badges
- 'Refresh' (re-fetch cache, no rebuild) + '立即重建' (manual
  stability rebuild) buttons
- six class groups (Identity / Style / Tooling / Veto / Goal /
  Channel) in the same render order as the Rust `CLASS_RENDER_ORDER`,
  with a Chinese + English label + one-line description per class
- one row per facet: name:value, state badge (active / provisional /
  candidate / forgotten with semantic colours), stability, evidence
  count, last-seen timestamp, per-row dismiss button (calls
  `memory_learning_dismiss_facet` and optimistically flips the state
  to 'forgotten' so the row stays but dims)
- empty state when the cache is fresh, dashed-border card
- inline structured error when learning_enabled=false

Wire changes:
- ui/src/atoms/settings-tab.ts: add 'learnedProfile' to SettingsTab
- ui/src/components/settings/SettingsNav.tsx: add the new entry under
  the 核心 group with the UserCircle2 icon
- ui/src/components/settings/SettingsPanel.tsx: switch + label
- ui/src/lib/types.ts: add FacetDto, LearningListFacetsInput,
  LearningRebuildNowInput, LearningDismissFacetInput,
  LearningDismissOutcome
- ui/src/lib/tauri-bridge.ts: three wrappers for the IPC commands

Tests:
- LearnedProfileTab.test.tsx — 5 tests: empty state, grouped render,
  dismiss optimistic flip, rebuild triggers IPC + re-fetch, disabled
  error surfaces inline
- SettingsNav.test.tsx — added '学到的偏好' to the spot-check labels
  and dropped the now-misleading '9 tabs' wording

Verification: `npx tsc --noEmit` clean. Vitest must run Mac-side; the
component follows the same shape as MemoryHealthPanel which has been
green in CI since Phase 4.
