feat(learning): wire chat-turn extractor into agent dispatcher (Sprint 2.0)

The dormant pipeline disclosed in Sprint 1's summary is now active.
Before this commit:
- producer (`extract_from_chat_turn`) had no callsite
- consumer (`set_learned_profile_block`) had no callsite
- FacetCache stayed empty + PROFILE block never injected even when
  candidates were manually pushed via IPC

This commit wires both halves into all three ChatDelegate::new()
callsites in tauri_commands.rs:

1. `send_message` (chat IPC, line 515)
2. `send_agent_message` (agent IPC, line 8248 — inside tokio::spawn)
3. `start_agent_teams` (teams orchestrator factory closure, line 11014)

ChatDelegate (`agent/dispatcher.rs`):
- 5 new fields: learning_buffer / learning_llm / learning_enabled /
  learning_db / learning_llm_daily_budget
- `set_learning_pipeline` setter wires them from AppState
- `before_llm_call` at iteration=0 finds the latest user message
  text, spawns the extractor, and pushes 0+ candidates into the
  shared buffer. Failures are logged + swallowed inside the
  spawned task — producer bugs can never stall the LLM call.

AppState (`app.rs`):
- New `learning_llm: Option<Arc<dyn MemoryOsLlm>>` field.
- Built at bootstrap when `memory_os.learning_enabled = true` AND
  `learning_llm_daily_token_budget > 0`. Shares the same
  `MemoryOsLlmClient` adapter used by wiki/lint/entity.

Sprint 2.1b — Daily token cap (also landed in this commit):
- New `memory_os.learning_llm_daily_token_budget` (default 30_000).
- New `cost_store::today_learning_tokens(db)` helper sums
  `cost_records.model LIKE 'memory_learning%' AND created_at >= today`.
- Dispatcher calls this before invoking the LLM layer; over-budget
  -> LLM skipped (regex layer is free + always runs).

Adjacent change (not scope creep): `current_day_start_ms` is added
to cost_store as the natural pair of `current_month_start_ms`. It
makes both per-day caps (Sprint 2.1b learning, future per-day lint
override) share the same UTC-day calculation.
