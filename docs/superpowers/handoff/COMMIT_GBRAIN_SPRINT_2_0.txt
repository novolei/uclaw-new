feat(gbrain): bundle pipeline for Bun + gbrain source (Sprint 2.0)

Sprint 2.0 of the gbrain integration track (Path C-2 — bundle Bun
runtime + gbrain source as Tauri resources, spawn at runtime as a
stdio MCP subprocess). This commit lays the build-pipeline plumbing.
Sprint 2.1 will register the actual MCP server entry in
`mcp_servers.json` defaults so it auto-spawns at boot via the PR-1
auto-connect hook.

## What's bundled

Two new gitignored directories under `src-tauri/`:

- `bunembed/bun` — Bun static binary (~50MB/platform), downloaded by
  `setup-bun-runtime.sh`. Picks `bun-{darwin,linux}-{aarch64,x64}` by
  uname. Mirrors `pyembed/` shape.
- `gbrain-source/` — `garrytan/gbrain` clone with `bun install
  --production`'d node_modules. Stripped `.git`. Cloned by
  `setup-gbrain-source.sh`.

Both declared as Tauri resources in `tauri.conf.json`:

```json
"resources": {
    "pyembed/python":          "python",
    "src/memu/memu_bridge.py": "memu_bridge.py",
    "../skills":               "skills",
    "bunembed/bun":            "bun",      // ← new
    "gbrain-source":           "gbrain"    // ← new
}
```

## Setup scripts

Both follow the existing `setup-python-env.sh` pattern — color
logging, `--clean`/`--yes`/`--help` flags, GitHub-API version
probe with hard-coded fallback, idempotent overwrite-or-skip.

`setup-bun-runtime.sh`:
- Env override: `BUN_VERSION` (default falls back to 1.1.42 if API
  unreachable)
- Detects platform via uname → picks the right release artifact
- Unzips to a tmpdir → copies bun binary to `bunembed/bun` →
  chmod +x
- Verifies installed version via `bun --version`

`setup-gbrain-source.sh`:
- Env overrides: `GBRAIN_REPO` (default `garrytan/gbrain.git`),
  `GBRAIN_REF` (default `main`), `PRUNE_DEV` (default on)
- Shallow clone (`--depth 1`); falls back to full-clone-then-checkout
  if shallow ref-resolution fails (e.g. for tags)
- Removes `.git` after clone to shrink bundle
- Runs the bundled `bunembed/bun install --production` against the
  clone, NOT system bun (so dev env doesn't need Bun pre-installed
  outside this repo)
- **PGlite verification step** — `bun build --compile` died on
  PGlite WASM ENOENT in Sprint 0; Path C-2 sidesteps that by
  keeping WASM at standard node_modules paths. The script now
  inspects `node_modules/@electric-sql/pglite/` after install,
  prints the WASM file inventory + sizes so the operator can
  confirm everything's where it needs to be before bundling.
- Probes for common entry-point paths (`src/index.ts`, `src/main.ts`,
  etc) and prints which one to use in Sprint 2.1's MCP config

## PGlite — the storage substrate

gbrain uses `@electric-sql/pglite` (PostgreSQL compiled to WASM)
as its storage backend. Path C-2's bundling approach handles it
naturally:

- `bun install` puts the WASM at standard node_modules path
- Tauri `resources` recurses, so the WASM gets bundled
- At runtime `bun gbrain-source/<entry>` resolves PGlite normally
- **Caveat**: PGlite needs a writable data dir at runtime. The
  Tauri resource dir is read-only on macOS (.app bundle). Sprint
  2.1 must pass a `PGLITE_DATA_DIR` (or gbrain's equivalent env)
  pointing at `~/.uclaw/gbrain/pgdata/` and `mkdir -p` it before
  spawn.

Estimated PGlite WASM size: ~3-5MB. Total `gbrain-source/` after
`bun install --production`: ~80-100MB (revised up from the
original 70MB estimate to account for node_modules + WASM).

## Verification

Sandbox can't run the actual download/install. Mac side:

```bash
./scripts/setup-bun-runtime.sh
./scripts/setup-gbrain-source.sh
# Confirm sizes look right (~50MB bunembed, ~70MB gbrain-source)
du -sh src-tauri/bunembed src-tauri/gbrain-source

# Confirm Tauri build picks them up
cd src-tauri && cargo tauri dev 2>&1 | grep -iE "resource|gbrain|bunembed" | head -20
```

Expected: Tauri logs include lines indicating `bun` + `gbrain`
resources are present in the bundle. Build doesn't error on missing
resources because the new entries match real paths.

If `garrytan/gbrain` URL or `main` ref needs tweaking:

```bash
GBRAIN_REPO=https://github.com/<your-fork>/gbrain.git \
GBRAIN_REF=v0.3.2 \
./scripts/setup-gbrain-source.sh
```
