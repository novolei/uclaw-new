fix(gbrain): init brain before seeding MCP entry (Sprint 2.1 init-fix)

## Why

Sprint 2.1 (#204) seeded the bundled gbrain MCP entry but never
initialized the brain itself. On every fresh launch, gbrain serve
exited immediately with "No brain configured. Run: gbrain init", and
uClaw's MCP manager logged "MCP server 'gbrain (bundled)' initialize
failed: Timeout" after the full 60s connect deadline. The agent's
tool table never saw mcp__gbrain__*.

## What

- Add `is_brain_initialized` (pure PG_VERSION probe) +
  `ensure_bundled_gbrain_initialized` (synchronous spawner) in
  src-tauri/src/mcp.rs.
- Stage 3 boot in src-tauri/src/main.rs now calls the spawner before
  seed_bundled_gbrain. First launch runs `bun gbrain init --pglite
  --yes` (~30-60s for ~63 migrations); warm launches short-circuit
  via PG_VERSION.
- Delete the pre-write of ~/.uclaw/gbrain/.gbrain/config.json that
  pointed at a pgdata/ dir gbrain init never uses (gbrain init writes
  its own config with the correct .gbrain/brain.pglite path).
- Rename seed_bundled_gbrain parameter `pgdata_dir → gbrain_home` to
  drop the misleading `.parent()` derivation hack.
- 4 new unit tests pin the PG_VERSION-probe contract + the spawner's
  idempotency short-circuit.
