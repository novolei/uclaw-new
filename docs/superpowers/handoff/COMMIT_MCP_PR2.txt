feat(mcp): refresh / ping / disconnect IPC + drawer affordances (PR-2)

Two new IPC commands wrap existing-but-orphaned McpManager methods,
and the detail drawer grows a secondary action row exposing them.
Quick-wins #4 and #5 from the 2026-05-18 audit.

## Backend (src-tauri/src/tauri_commands.rs)

- `refresh_mcp_tools(id: String) -> Vec<Value>` calls
  `McpManager::refresh_tools` (which already exists, mcp.rs:1125)
  and shapes the result the same way `list_mcp_tools` does. Useful
  when the user wants new tools without restarting the connection.
- `ping_mcp_server(id: String) -> u64` wraps
  `McpManager::ping_server` (mcp.rs:1114) and returns elapsed ms so
  the UI can show "Ping ✓ 23ms" rather than just success/fail.

Both registered in `main.rs` invoke_handler!.

## Bridge (ui/src/lib/tauri-bridge.ts)

`refreshMcpTools(id)` + `pingMcpServer(id)` wrappers. The
`disconnectMcpServer` wrapper was already there but had zero callers
(the audit's "orphaned IPC" finding); this commit gives it one.

## UI (ui/src/views/Kaleidoscope/modules/Integrations/McpDetailDrawer.tsx)

New secondary row below the existing 编辑/重启/移除 cluster, visible
only when `server.status === 'connected'`:

- 刷新工具 → `refreshMcpTools` → toast with new tool count
- 测试连接 → `pingMcpServer` → toast with elapsed ms
- 断开 → `disconnectMcpServer` → toast + parent refetch

Three independent busy flags (`refreshing` / `pinging` /
`disconnecting`) so the user can e.g. ping while a slow refresh is
still in flight. Errors fall through to `toast.error(String(e))`.

The drawer gains two optional `onToolsRefreshed` / `onDisconnected`
callbacks; `IntegrationsModule.tsx` wires them to its existing
`refetch()` so the tool list + status badge update without polling.

## Verification

- `npx tsc --noEmit` clean in sandbox
- cargo build + cargo test mcp:: must run Mac-side

Behaviour to spot-check after merge:
1. Open a connected MCP server's detail drawer — three new ghost
   buttons appear in a second row.
2. Click "测试连接" — should show "Ping ✓ NNms" within a few seconds.
3. Click "刷新工具" — toast reports the tool count; the tool list
   above re-renders with any newly-discovered tools.
4. Click "断开" — server flips to disconnected status, the secondary
   row hides itself.
