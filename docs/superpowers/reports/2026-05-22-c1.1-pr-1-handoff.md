## Summary

Wires Bundle 17-A's `fold_diff` module (previously dead code) into
the `/compact` user-message intercept. When a prior `StructuredFold`
baseline exists for the session AND the new fold's drift is below
`memubot_config.context.fold_delta_threshold` (default 5), the
`/compact` placeholder becomes the byte-stable prior-fold markdown
plus a `<context_changes_since_last_fold>` block — instead of a
fresh full re-render. The byte-stable prefix lets the next turn's
M2-I prompt-cache breakpoint hit a stable prefix; cached_input_tokens
should rise on subsequent turns.

Closes M2 closeout sub-task C1.1 PR-1 per
[`docs/superpowers/queue/C1-execution-queue.md`](../docs/superpowers/queue/C1-execution-queue.md);
drives M2-D Diff-based re-injection from ~70% to ~85%; unblocks
C1.1 PR-2 (Bundle 17-C telemetry) and task #146.

## Behavior coverage

- first compact → FullRewrite
- drift in (0, threshold) → DeltaRendered{drift}
- drift == threshold → FullRewrite (strict less-than)
- drift > threshold → FullRewrite
- drift == 0 → FullRewrite (no "nothing changed" block)
- changed-axis rationale flip routes to delta path with prior+now visible
- delta placeholder bytes start with `prior.to_markdown()` byte-for-byte
  (the prompt-cache stability invariant — the whole point of the path)

- multi-axis delta covers all 8 axes
